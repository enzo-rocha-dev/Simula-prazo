# 📦 Simulador de prazos comerciais

## 📌 Contexto

Esse projeto surgiu de uma necessidade real dentro da área comercial da atlética da faculdade. No dia a dia, era comum ter dúvidas como:

- Se eu fizer o pedido hoje, quando chega? 📅
- Se preciso que chegue em uma data específica, quando devo pedir? ⏳

Sem uma ferramenta clara, essas decisões acabavam sendo feitas no “feeling”, o que gerava erros e retrabalho.

---

## 🎯 Objetivo

Criar uma ferramenta simples para simular prazos de forma rápida e visual, ajudando na tomada de decisão do comercial.

A ideia foi manter tudo o mais direto possível:
- sem complexidade desnecessária
- fácil de usar
- fácil de acessar

---

## ⚙️ Desenvolvimento

Esse projeto foi feito utilizando **vibecoding**, com foco em construir algo funcional rapidamente.

Todo o desenvolvimento foi feito **utilizando o ChatGPT** 🤖, desde a estrutura inicial até os ajustes de lógica e interface.

Ao longo do processo:
- comecei com uma ideia simples
- fui iterando melhorias
- ajustando comportamento com base no uso real

Inicialmente pensei em usar Python (Back-End) + estrtura visual em HTML (Front-End), mas optei por uma solução **100% em HTML + JavaScript**, pois:
- simplifica o deploy (principalmente para o github pages, que era meu objetivo)
- funciona direto no navegador
- resolve completamente o problema

---

## 🧠 Funcionalidades

- Simulação de prazo a partir de uma data 📅
- Cálculo reverso (quando pedir para chegar em uma data) 🔄
- Suporte a:
  - dias corridos
  - dias úteis
  - dias úteis com sábado
- Consideração de feriados
- Visualização em calendário:
  - 3 meses
  - semanas iniciando na segunda
  - destaque do período calculado (cores)

---

## 🚀 Resultado

A ferramenta resolve de forma direta o problema de planejamento de prazos, permitindo:

- mais segurança nas decisões
- menos erro operacional
- visão clara do período

---

## 💡 Observação

Esse projeto não tem como foco complexidade técnica, mas sim **resolver um problema real de forma simples e eficiente**.

Mostra como, com uma boa definição de problema e uso de ferramentas certas, é possível construir algo útil rapidamente.
